# Online Style Research Notes

Use these sources as a starting point when the user asks Codex to research system-map visual styles. The goal is to extract reusable style rules, not to copy exact layouts, icons, screenshots, or proprietary templates.

## Search Scope

Use Pinterest/image search first when requested, then broaden to public sources if Pinterest is blocked by login or crawling limits.

Recommended queries:

- `stakeholder map design`
- `service system map`
- `ecosystem map design`
- `value network map`
- `systems mapping portfolio`
- `service design blueprint stakeholder`
- `design strategy system diagram`

## Reference Types Observed

### Concentric stakeholder maps

- Example sources: Medium stakeholder map article by Juan Gallardo; Netguru design sprint stakeholder map; NHS/GOV.UK stakeholder mapping article.
- Reusable rules: center the user/product/service; use rings for closeness, importance, or influence; use short labels and a small legend; keep outer actors visually lighter.
- Do not copy: exact icons, example stakeholder names, or specific ring labels unless they fit the user's project.

### Power-interest maps

- Example source: Lucid stakeholder map with icons.
- Reusable rules: use axes or quadrants for influence and interest; reduce relationship lines; use quadrant labels as the main structure.
- Do not copy: template-specific icon art or exact color blocks.

### Layered service system maps

- Example sources: Methods Digital public sector service design article; Mike Laurie systems-thinking service blueprint article.
- Reusable rules: divide the system into layers; show user needs, service landscape, capabilities, technology, operations, or support processes as bands; use thin connectors and compact cards.
- Do not copy: the specific organization framework or article-specific service labels.

### Value network and ecosystem maps

- Example source: design@tive value network/ecosystem map article.
- Reusable rules: use edge types and icons/labels to distinguish value, money, information, and service flows; include non-human actors such as platforms and infrastructure when relevant.
- Do not copy: exact demonstration graphic or icon set.

### Editorial portfolio maps

- Common source pattern: Behance, portfolio boards, Figma community diagrams, and design strategy posts.
- Reusable rules: lower the node count, use more whitespace, make the title and legend part of the composition, and show only the relationships needed for the page's argument.
- Do not copy: signature poster layouts, branded graphics, or distinctive illustration styles.

## How to Apply

For each user project:

1. Choose one of the style families in `style-library.md`.
2. Borrow only general rules: composition, label density, line language, palette logic, hierarchy, legend treatment.
3. Keep stakeholder analysis independent from the style source.
4. If the source is not available or cannot be cited, mark the style direction as inferred from visual research rather than sourced.
